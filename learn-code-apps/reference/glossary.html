Reference · Power Platform Code Apps

Glossary

The course's fixed vocabulary. Every lesson uses these terms exactly as defined here.

The app and its runtime

Code appA single-page web app you write in your own IDE (React/TypeScript + Vite in the templates) that is hosted and delivered by the Power Apps host, with data access brokered through connectors. GA since 2026-02-05. overview
Power Apps hostThe platform runtime that serves your compiled bundle, signs the user in with Entra ID, enforces governance (DLP, Conditional Access), and hands your code a context via the SDK. What you don't build is what the host provides. architecture
@microsoft/power-apps SDKThe npm client library that performs the host handshake (getContext()) and executes data operations against connectors from your app code. npm
power.config.jsonThe file pac code init creates in your repo; identifies the app to the platform (name, build path, entry point).
Content Security Policy (CSP)The browser policy the host applies to your app. Default is restrictive: connect-src 'none' (no external calls, including telemetry) and worker-src 'none' (no service workers → no offline/PWA). Admins allow-list exceptions per environment. limits

Data

ConnectorA platform-managed proxy to a data source or API (Dataverse, Azure SQL, SharePoint, …). The only data path a code app has — there are no direct database connections. connect-to-data
Custom connectorA connector you define over your own REST API (e.g., Azure Functions). The escape hatch for real business logic — and the exit-portability boundary.
ConnectionA user's authorized instance of a connector (their credentials/consent). Apps share connector definitions; each user runs on their own connection.
DataversePower Platform's built-in relational data service. Not required to run a code app, but required for Dataverse data sources and solution-based ALM. connect-to-dataverse
DelegationWhether a query executes at the data source (delegable) or client-side over a partial download (non-delegable). Non-delegable queries silently truncate past 500 rows (max 2,000) — the single biggest behavioral trap for SQL-minded developers.
Power Platform Request (PPR)The metered unit of platform usage. Every connector call counts against a per-user daily quota: 40,000 (Premium) vs 6,000 (per-app/PAYG/M365). request limits

Licensing

Power Apps PremiumThe per-user license ($20/user/mo list; $12 at 2,000+ seats) every code-app end user needs. Unlike canvas apps, there is no standard-connector discount. prerequisites
Per-app pay-as-you-go$10 per active user per app per month, metered. Documented for Power Apps generally; its validity for code apps specifically is unconfirmed — flagged as a gap.
Seeded (M365) rightsThe limited Power Apps entitlement inside Microsoft 365 licenses: standard connectors + Dataverse for Teams only. Does not cover code apps.
Developer PlanFree plan giving developers up to 3 personal Dataverse environments for building and testing. Building costs nothing; only end users need licenses. dev environments

ALM and governance

EnvironmentThe platform's isolation unit (dev/test/prod each get one). Code app operations are off by default per environment — an admin toggle enables them (confirmed empirically, 2026-07-21).
SolutionPower Platform's packaging/transport unit for ALM. Code apps attach to one at push (pac code push --solutionName); solutions move between environments via export/import or Pipelines. push-to-solution
Connection referenceA solution-aware pointer to a connection, rebindable per environment — how one artifact targets dev data in dev and prod data in prod.
Power Platform PipelinesMicrosoft's built-in solution deployment system (dev→test→prod). The first-party documented CI/CD path for code apps; using Azure DevOps/GitHub at the solution layer instead is possible but community-documented.
Managed EnvironmentAn admin-enabled environment tier with extra governance (sharing limits, policies). Whether code apps strictly require it is unconfirmed — treat as likely.
DLP policyData Loss Prevention: admin rules classifying which connectors may be used together. Enforced against code apps at launch and runtime — inherited governance your self-hosted apps never had.
pac CLI / power-apps npm CLIThe developer tooling. The pac code command group is preview-and-deprecating; its replacement, the power-apps npm CLI, is itself preview. Live churn — pin versions. CLI reference
