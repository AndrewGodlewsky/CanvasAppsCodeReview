Lesson 0001 · Lead the Tier

The code app mental model: what runs where

One tangible win: after this lesson you can draw the code-app runtime on a whiteboard and say, without notes, what the platform owns and what your code owns.

Why this is lesson one

Every conversation you'll lead — with an executive asking "why are we paying for this?", an engineer asking "where does my code run?", or a security reviewer asking "who serves the bundle?" — starts from this picture. Licensing, limits, and ALM (lessons 3–7) are all consequences of it.

The picture

Your repo (you own this)React/TypeScript source · Vite build · your Git, PR review, CI, security scanning — unchanged.
⬇ pac code push — uploads the compiled bundle only, one-way; the platform never stores source
Power Apps host (platform owns this)Serves your bundle · signs users in with Entra ID · enforces DLP, Conditional Access, sharing, CSP · meters usage.
⬇ user opens the app · ⬆ getContext() handshake via the SDK
User's browserRuns your SPA. All of your code executes here — there is no server tier of yours.
Connector layerEvery data call goes through a connector, metered as Power Platform Requests.
DataDataverse · Azure SQL · SharePoint · your own APIs behind custom connectors.

Dashed boxes: yours. Solid boxes: the platform's. The whole adoption trade lives in that dashed/solid split.

Five facts that anchor everything

1Your code runs only in the user's browser. A code app is a client-side SPA; anything that can't live in a browser must move behind a connector. [arch]
2The host is the value proposition. Hosting, Entra sign-in, governance, delivery — that's what you stop building versus React + App Service. [overview]
3Data is connector-only. No connection strings, no direct DB. Typed clients are generated per data source; queries inherit delegation semantics, not SQL semantics. [data]
4Git is the only home your source has. Push is one-way and compiled-only; there is no download. Lose the repo, lose the app. [cli]
5Every end user needs Power Apps Premium. No standard-connector discount, no M365 seeded coverage. (Our tier works because we already hold Premium broadly.) [prereqs]
ADR tie-in. Fact 1 is why the tier ADR has a logic-placement policy at all: with no server tier of ours inside the platform, real business logic either degrades into flows/plugins or moves to our own Azure services behind custom connectors. The ADR chose the latter as the default.
