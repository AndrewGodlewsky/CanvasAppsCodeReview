# Pro-code baseline established; research exposure is not retention

The learner is a professional React/TypeScript developer in a mature DevOps org (Git, PR review, CI/CD) — teach nothing about general programming, source control, or pipelines. They co-produced four research briefs and a tier-adoption ADR on 2026-07-21, so all code-apps vocabulary has been *seen* once, but only via summaries — treat it as exposure, not storage strength, and verify with retrieval before building on any of it.

**Implications:** start teaching at platform semantics (host runtime, connector brokering, delegation, licensing gates), not at React or tooling. The ADR gives every lesson a real-world anchor; use it.
